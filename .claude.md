# Playwright 練習用プロジェクト

このプロジェクトは、React + TypeScript + Playwright を使用したE2Eテストの学習・練習用アプリケーションです。

## プロジェクト概要

- **フレームワーク**: React 18 + TypeScript
- **ビルドツール**: Vite
- **テストフレームワーク**: Playwright
- **目的**: Playwrightを使ったE2Eテストの学習と実践

## コーディング規約

### 言語設定
- **コメント**: すべて日本語で記述
- **変数名・関数名**: 英語のキャメルケース
- **インターフェース名**: パスカルケース

### TypeScript
- 型定義を明示的に行う
- `any`型の使用は避ける
- インターフェースを活用する

### React
- 関数コンポーネントを使用
- フックスを活用する
- propsの型定義を行う

### Playwright
- Page Object Modelパターンを推奨
- わかりやすいテスト名（日本語OK）
- アクセシビリティロケーター（role、label等）を優先的に使用

## ディレクトリ構造

```
playwright-practice-app/
├── src/                 # ソースコード
│   ├── components/      # Reactコンポーネント
│   ├── pages/           # ページコンポーネント
│   └── utils/           # ユーティリティ関数
├── tests/               # Playwrightテスト
│   ├── e2e/            # E2Eテスト
│   └── fixtures/       # テストフィクスチャ
├── public/              # 静的ファイル
└── playwright.config.ts # Playwright設定
```

## よく使うコマンド

```bash
# 開発サーバー起動
npm run dev

# ビルド
npm run build

# Playwrightテスト実行
npx playwright test

# Playwright UIモードでテスト実行
npx playwright test --ui

# 特定のブラウザでテスト実行
npx playwright test --project=chromium

# テストレポート表示
npx playwright show-report

# Playwright Codegen（テスト自動生成）
npx playwright codegen http://localhost:5173
```

## Claude Code使用時の注意事項

1. **テストファイル作成時**
   - テスト名は日本語で記述可能
   - Page Object Modelを活用
   - アサーションは明確に

2. **コンポーネント作成時**
   - TypeScriptの型定義を必ず含める
   - propsインターフェースを定義
   - コメントは日本語で

3. **バグ修正時**
   - 変更内容を日本語コメントで説明
   - テストケースも併せて更新

4. **リファクタリング時**
   - 既存のテストが通ることを確認
   - 型安全性を保つ

## 学習の進め方

1. 基本的なテストの書き方を学ぶ
2. Page Object Modelを実践する
3. さまざまなロケーター戦略を試す
4. 並列実行とテスト分離を理解する
5. ビジュアルリグレッションテストを試す
6. API モックとインターセプションを学ぶ
